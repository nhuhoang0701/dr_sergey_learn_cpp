# Use Ninja as a fast build backend for CMake

**Category:** Tooling & Debugging  
**Item:** #257  
**Reference:** <https://ninja-build.org>  

---

## Topic Overview

Ninja is a small, fast build system designed for maximum speed. CMake can generate Ninja build files instead of Makefiles.

| Feature | Make | Ninja |
| --- | --- | --- |
| Startup overhead | High (re-parses Makefile) | Minimal (binary format) |
| Dependency tracking | File-based | Precise dependency log |
| Parallel scheduling | Manual `-j N` | Smart auto-parallelism |
| Incremental rebuilds | Good | Excellent |
| Typical speedup | Baseline | 2-10x faster on large projects |

## Self-Assessment

### Q1: Configure CMake with Ninja and compare build times

```bash

# Step 1: Install Ninja
# Ubuntu: sudo apt install ninja-build
# macOS:  brew install ninja
# Windows: choco install ninja  (or download from ninja-build.org)

# Step 2: Configure with Ninja generator
$ cmake -G Ninja -B build-ninja -DCMAKE_BUILD_TYPE=Release
$ cmake -G "Unix Makefiles" -B build-make -DCMAKE_BUILD_TYPE=Release

# Step 3: Build and time both
$ time cmake --build build-ninja
# real    0m4.2s   <-- Ninja

$ time cmake --build build-make -- -j$(nproc)
# real    0m12.8s  <-- Make (even with -j parallel)

# Why Ninja is faster:
# 1. No Makefile re-parsing on each invocation
# 2. Minimal stat() calls (uses .ninja_deps log)
# 3. Better parallel scheduling of independent targets
# 4. Pools limit resource contention (e.g., linking)

```

```cmake

# CMakeLists.txt — works the same with any generator
cmake_minimum_required(VERSION 3.20)
project(FastBuild LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 20)

# Add many source files:
file(GLOB_RECURSE SOURCES src/*.cpp)
add_executable(app ${SOURCES})

# CMake presets (CMakePresets.json) for team-wide Ninja:
# {
#   "configurePresets": [{
#     "name": "default",
#     "generator": "Ninja",
#     "binaryDir": "build"
#   }]
# }

```

### Q2: Control parallelism and understand Ninja's scheduling

```bash

# Control parallelism:
$ ninja -C build-ninja -j8    # use 8 parallel jobs
$ ninja -C build-ninja -j1    # sequential (for debugging build issues)
$ ninja -C build-ninja        # auto-detect cores (default)

# Ninja scheduling:
# 1. Reads build.ninja (binary-like format, fast parse)
# 2. Builds dependency graph
# 3. Starts with leaf nodes (no dependencies)
# 4. As each job finishes, immediately starts next ready job
# 5. Limits link jobs via "pool" to avoid memory exhaustion

# View dependency graph:
$ ninja -C build-ninja -t graph | dot -Tpng > deps.png

# Show build targets:
$ ninja -C build-ninja -t targets all

# Dry run (show what would be built):
$ ninja -C build-ninja -n

# Verbose (show actual commands):
$ ninja -C build-ninja -v

# Build timing:
$ ninja -C build-ninja -j8 2>&1 | tee build.log
$ grep -E 'real|user|sys' build.log

```

```ninja

# Example build.ninja snippet (generated by CMake):
rule CXX_COMPILER
  command = g++ -std=c++20 -O2 -o $out -c $in
  depfile = $out.d

build src/main.o: CXX_COMPILER src/main.cpp
build src/util.o: CXX_COMPILER src/util.cpp

# Link pool limits concurrent link jobs:
pool link_pool
  depth = 2

rule LINK
  command = g++ -o $out $in
  pool = link_pool

build app: LINK src/main.o src/util.o

```

### Q3: Export `compile_commands.json` for clangd

```bash

# Generate compile_commands.json with Ninja:
$ cmake -G Ninja -B build \
    -DCMAKE_EXPORT_COMPILE_COMMANDS=ON \
    -DCMAKE_CXX_STANDARD=20

# Creates: build/compile_commands.json
# Example content:
# [
#   {
#     "directory": "/home/user/project/build",
#     "command": "g++ -std=c++20 -O2 -I/home/user/project/include -o ... -c ../src/main.cpp",
#     "file": "/home/user/project/src/main.cpp"
#   },
#   
# ]

# Symlink to project root for clangd:
$ ln -s build/compile_commands.json .

# clangd reads this for:
# - Include path resolution
# - Compiler flags (-std=c++20, -D defines)
# - Target-specific options
# - Code completion, go-to-definition, diagnostics

# CMake preset with compile commands:
# CMakePresets.json:
# {
#   "configurePresets": [{
#     "name": "dev",
#     "generator": "Ninja",
#     "cacheVariables": {
#       "CMAKE_EXPORT_COMPILE_COMMANDS": "ON"
#     }
#   }]
# }

```

## Notes

- Ninja is the default generator for Android NDK and Chrome builds.
- Use `ninja -t compdb` to regenerate compile_commands.json.
- Ninja Multi-Config: `cmake -G "Ninja Multi-Config"` for Debug/Release in one build dir.
- `ninja -t clean` removes all build outputs.
- On Windows, use Ninja with MSVC via `vcvarsall.bat` + `cmake -G Ninja`.
